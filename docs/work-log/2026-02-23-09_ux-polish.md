### 各実装の設計メモ

**AppShell:**
- `usePathname()` で `/login` か判定し、NavBar と SaveStatusIndicator を条件付き表示
- `main` の `padding-top` も `/login` では除去（全画面ログインカード用）
- layout.tsx を Server Component のまま維持するため、条件分岐ロジックはこちらに分離

**TableSkeleton:**
- `.shimmer` クラス: `background-size: 200%` + `animation: shimmer 1.5s infinite` で光が左→右に流れる
- 週ヘッダー・日付ラベル・4行のプレースホルダー行で構成
- log/retro 共通の汎用コンポーネント（列幅は厳密に一致させない）

**ローディング状態:**
- ページヘッダーもシマーバーでスケルトン表示
- `TableSkeleton` を2個並べて「2週分のデータ読み込み中」を表現

**エラー状態:**
- `useLog` / `useRetro` が `retry` 関数を返す（内部では `mutate()` を呼ぶ）
- `error` が truthy のときにエラーメッセージ + 再試行ボタンをインライン表示

**テーブル横スクロール（WBS 9.1）:**
- テーブルラッパーに `overflow-x-auto` を追加
- 内側の `div` に `min-w-[480px]`（log）/ `min-w-[360px]`（retro）で最小幅を確保

**ログイン画面（WBS 6.1）:**
- モック時はフォーム送信で `/log` へ直接遷移（認証なし）
- 本番実装時は `handleSubmit` に Supabase Auth の呼び出しを追加するだけでよい設計

**WBS 8.x について:**
- `export.ts`（buildLogText / buildRetroText）は WBS 3.x 時点で実装済み
- WeekGroup への組み込みも log/retro ページで `onCopy` として渡し済み
- 追加実装不要、WBS 8.x は完了扱い

## 決定事項

- shimmer は globals.css のプレーン CSS クラスとして定義（Tailwind の `@keyframes` に乗せない）
- ログインページのNavBar非表示は route group ではなく `AppShell` クライアントコンポーネントで対応（既存ファイルの移動が不要）
