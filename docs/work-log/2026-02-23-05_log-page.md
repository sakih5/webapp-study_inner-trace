### useLog フックの設計

`useSWRInfinite` を使い、2週単位でページングする：

```
pageIndex=0: 今週〜2週前
pageIndex=1: さらに2週前
...
```

- 前ページが空配列を返したら key 関数が `null` を返し、それ以上ロードしない
- `update` は楽観的更新（即時ローカル反映 → API 失敗時に `mutate()` でロールバック）
- `add` / `remove` は更新後に `mutate()` で全ページ再検証

### LogRow の設計

- 時間 select は `generateTimeOptions()` で生成（05:00〜29:50、10分刻み）
- 24:00 以上は表示上 `翌HH:MM`、保存値は `HH:MM` のまま
- 感情の孤立値（削除済みオプション）: `user_options` に存在しない場合、先頭に `{value}（削除済み）` をグレー表示で追加
- 削除ボタンは `group-hover` で表示（通常時は非表示）

### 仮行（provisional row）フロー

```
＋ ボタン押下
  → provisional state にセット（date=今日, time=null, emotion=null）
  → allEntries に id='__provisional__' の LogEntry として混入
  → sortLogEntries が '__provisional__' を日付グループ先頭に配置
  → LogRow の autoOpen=true で action セルが自動展開・フォーカス
  → action に入力して blur or debounce
      ├─ 入力あり → api.post → add() → provisional を null にクリア
      └─ 空 → provisional を null にクリア（APIコールなし）
```

仮行の date/time/emotion 変更は `handleProvisionalUpdate` でローカルステートのみ更新。POST 時に一緒に送信される。

### 無限スクロール

const observer = new IntersectionObserver(

センチネル div はページ末尾に配置し、ロード中はスピナーテキストを表示。

```
WeekGroup (groupByWeek で週単位に集約、最新週が defaultOpen=true)
  └ DateGroup (groupByDate で日付単位に集約、降順ソート)
      └ テーブル (LogRow × N)
```

コピーボタンは仮行（`__provisional__`）を除いた実エントリのみ `buildLogText` に渡す。

## 決定事項

- `sortLogEntries` は id が `'__provisional__'` のエントリを同日グループ内で強制的に先頭に置く
- 感情オプションはページ側で `useSWR('/settings/options?type=emotion')` で取得し、`LogRow` に props として渡す（LogRow 内で独自フェッチしない）
- テーブルは `table-fixed` で固定幅カラムを使い、行動セルが残り全幅を取る
