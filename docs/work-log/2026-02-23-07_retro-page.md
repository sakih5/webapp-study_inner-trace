### log との構造上の差分

| 項目 | /log | /retro |
|------|------|--------|
| カラム | Start / End / Action / Emotion | Type / Category / Content |
| 仮行POSTトリガー | `action` に入力 | `content` に入力 |
| ソート | 日付降順 → 同日内は開始時間降順 | 日付降順のみ |
| 孤立値対応 | emotion（null許容） | type / category（両方必須） |
| 無限スクロール | あり（2週単位） | あり（同じ仕組み） |

### 設計メモ

- DATE 列は行動記録と同様に除外。日付グループラベルで代替
- `sortRetroEntries` は `sortLogEntries` と同じ `__provisional__` 先頭ロジックを適用
- 仮行の type / category は `typeOptions[0]` / `categoryOptions[0]` をデフォルト値として使用
  - 追加ボタン押下時点でオプションが未ロードの場合、`content` 入力時の POST で再参照する
- RetroRow の type / category select は `null` 非許容（`—` 未選択オプションなし）

## 決定事項

- type / category は必須フィールドのため未選択オプション（`—`）を設けない
- 孤立値（削除済みオプション）は emotion と同じルールでグレー表示
- `sortRetroEntries` は `logUtils.ts` に追記（retro専用ファイルを作らない）
